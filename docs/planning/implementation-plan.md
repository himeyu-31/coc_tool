## リスクと対策
- データ仕様の増加: JSONスキーマを先に固定し破壊的変更を避ける
- データ量の多さ: 機能実装と分離してマスタ整備タスクを持ち、優先データから段階投入する
- UI複雑化: ウィザードを維持し1画面1目的を守る
- 出力不整合: CSV/XLSXのスナップショット確認を毎週実施
- 機能過多: 「GM向け」「PL向け」の入口で役割を分離し、初見で迷わせない
- ルール差分の混乱: v1はCoC7版を明記し、6版対応は別フェーズに分離する
- セッションデータ肥大化: キャラ固有データとセッション固有データの保存先を分ける
