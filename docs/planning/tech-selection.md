# 技術選定表

## 結論
- 採用: Webアプリ（PWA対応）
- 理由: 配布しやすい、更新しやすい、CSV/XLSX連携が容易
- 補足: 先駆者の共通仕様を踏襲しつつ、CoC7版特化の早引き・管理機能で差別化する

## 比較表

| 項目 | Web + PWA（推奨） | Flutter | Electron/Tauri |
|---|---|---|---|
| 学習コスト | 低〜中 | 中 | 中 |
| 初期開発速度 | 速い | 中 | 中 |
| 配布 | URLで即配布 | ストア/配布導線が必要 | インストーラ配布 |
| 継続更新 | 非常に高い | 高い | 高い |
| スプシ/Excel連携 | 容易 | 容易 | 容易 |
| 総合 | 最適 | モバイル特化なら有力 | PC特化なら有力 |

## 採用スタック（MVP）
- Next.js
- TypeScript
- React
- MUI
- Zustand
- React Hook Form + Zod
- SheetJS（XLSX） + CSV出力

## 競合を踏まえたプロダクト方針
- 共通仕様として維持するもの:
	- ブラウザ完結
	- スマホ対応
	- キャラシ保存、再編集
	- 自動計算
	- CSV/XLSXなどの外部出力
- 差別化として強めるもの:
	- CoC7版専用データモデル
	- 職業詳細の早引き
	- 狂気の発作ロール早引き
	- 武器ダメージ早引き
	- 初心者向け説明ページ
	- キャラシ管理、SAN計算、ダメージ計算の一体化

## データ/保存方針
- 初期: JSON定義 + ブラウザローカル保存
- 将来: Supabase/Firebaseを追加してアカウント同期対応

## 拡張方針
- 追加機能は「モジュール」単位で拡張
- 例: 武器の早引き表、狂気ロール早引き、呪文参照、所持品テンプレート
- データ追加中心で機能追加できる構造にする

## 情報設計の前提
- ルールデータを以下で分離する
	- 職業データ
	- 技能データ
	- 武器データ
	- 狂気表データ
	- 初心者向け解説データ
- 画面は以下の3系統に分ける
	- 作成
	- 早引き
	- 管理
