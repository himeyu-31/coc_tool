# 開発ドキュメント案内

このフォルダは、企画判断・画面設計・実装計画・タスク分解を分けて管理するための作業用ドキュメント置き場です。

## まず読む順番

1. implementation-plan.md
   - 現在の進捗と、どこまで実装済みかを確認するための主ドキュメント
2. issue-breakdown.md
   - 残タスクを Epic / Issue 目線で分けて見るためのドキュメント
3. enhancement-requests.md
   - 直近の改善要望を、実装論点と優先度に分けて確認するためのドキュメント
4. profession-formula-inventory.md
   - 現在の職業一覧、ポイント式、技能構成を修正前提で確認するためのドキュメント
5. ui-flow.md
   - 画面遷移や利用導線を確認したいときに使う
6. data-model.md
   - データ構造や責務の境界を確認したいときに使う
7. tech-selection.md
   - 技術選定の理由を振り返りたいときに使う
8. wireframes.md
   - 初期レイアウト検討の履歴
9. competitive-analysis.md
   - 参照した競合・既存サービスの観点整理

## 現在の実装状況の要約

- キャラクター作成ウィザードは CoC7 前提で実装済み
- キャラクター管理では CSV / XLSX の出力と再取込まで実装済み
- 職業、技能、武器、狂気表、用語集の早引き UI 初版は実装済み
- 初心者ガイド本文は実装済み
- ダークモード、レイアウト幅調整、文言整理などの UI 改善も反映済み
- 狂気表と用語集のマスタは拡充済みで、用語集は関連用語ジャンプに対応済み
- GitHub Pages で初版公開確認済み

## 運用ルール

- 進捗の更新は、まず implementation-plan.md を正とする
- タスク粒度の整理や実施順の見直しは issue-breakdown.md へ反映する
- 仕様の背景や意思決定は、対応する個別ドキュメントへ追記する
- 実装が進んで古くなった記述は、プレースホルダ表現を残さず更新する
